# 12 - Database Migration SQL

## 1. Tujuan Dokumen

Dokumen ini berisi rancangan SQL migration untuk database **ProjectPilot AI** menggunakan **Supabase Postgres**.

Database ini dirancang untuk mendukung:

1. Authentication berbasis Supabase Auth.
2. Multi-workspace.
3. Role-based access.
4. Project management.
5. Task management.
6. Kanban workflow.
7. AI generation log.
8. Reports.
9. Discord, Telegram, dan Gmail integrations.
10. Activity log.
11. Notification log.
12. Row Level Security.

---

# 2. Migration Strategy

Agar rapi dan mudah dimaintain, jangan buat satu file SQL raksasa.

Gunakan pembagian migration seperti ini:

```txt
supabase/migrations/
├── 0001_extensions_and_helpers.sql
├── 0002_core_tables.sql

├── 0004_rls_helper_functions.sql

```

## 6.1 Check Workspace Member

```sql
create or replace function public.is_workspace_member(target_workspace_id uuid)
returns boolean
language sql
security definer
set search_path = public
as $$
  select exists (
    select 1
    from public.workspace_members
    where workspace_id = target_workspace_id
      and user_id = auth.uid()
      and status = 'active'
  );
$$;
```

---

## 6.2 Check Workspace Role

```sql
create or replace function public.has_workspace_role(
  target_workspace_id uuid,
  allowed_roles text[]
)
returns boolean
language sql
security definer
set search_path = public
as $$
  select exists (
    select 1
    from public.workspace_members
    where workspace_id = target_workspace_id
      and user_id = auth.uid()
      and status = 'active'
      and role = any(allowed_roles)
  );
$$;
```

---

## 6.3 Check Project Access

```sql
create or replace function public.can_access_project(target_project_id uuid)
returns boolean
language sql
security definer
set search_path = public
as $$
  select exists (
    select 1
    from public.projects p
    join public.workspace_members wm
      on wm.workspace_id = p.workspace_id
    where p.id = target_project_id
      and wm.user_id = auth.uid()
      and wm.status = 'active'
  );
$$;
```

---

## 6.4 Check Task Access

```sql
create or replace function public.can_access_task(target_task_id uuid)
returns boolean
language sql
security definer
set search_path = public
as $$
  select exists (
    select 1
    from public.tasks t
    join public.workspace_members wm
      on wm.workspace_id = t.workspace_id
    where t.id = target_task_id
      and wm.user_id = auth.uid()
      and wm.status = 'active'
  );
$$;
```

## 8.2 completed_at Trigger for Tasks

```sql
create or replace function public.set_task_completed_at()
returns trigger
language plpgsql
as $$
begin
  if new.status = 'done' and old.status is distinct from 'done' then
    new.completed_at = now();
  end if;

  if new.status is distinct from 'done' then
    new.completed_at = null;
  end if;

  return new;
end;
$$;

create trigger set_tasks_completed_at
before update of status on public.tasks
for each row execute function public.set_task_completed_at();
```

# 11. Important Security Notes

## 11.1 Service Role

`SUPABASE_SERVICE_ROLE_KEY` hanya boleh digunakan di server.

Tidak boleh:

```txt
NEXT_PUBLIC_SUPABASE_SERVICE_ROLE_KEY
```

## 11.2 Integration Config

Table `integration_settings` menyimpan config dalam JSONB.

Untuk MVP boleh digunakan, tetapi untuk production sebaiknya:

1. Encrypt token sebelum disimpan.
2. Mask token saat ditampilkan.
3. Jangan pernah kirim token ke frontend.
4. Sediakan disconnect/reset integration.

## 11.3 Gmail Token

Gmail access token dan refresh token harus dianggap rahasia.

Jangan ditampilkan di UI.

UI hanya menampilkan:

```txt
Connected as user@gmail.com
```

# 13. Migration Success Criteria

Database migration dianggap sukses jika:

1. Supabase project bisa menerima schema tanpa error.
2. User bisa register menggunakan Supabase Auth.
3. Profile user bisa dibuat.
4. Workspace bisa dibuat.
5. Creator bisa menjadi owner workspace.
6. Project bisa dibuat dalam workspace.
7. Task bisa dibuat dalam project.
8. Kanban status bisa berubah.
9. Project progress bisa dihitung dari task.
10. AI generation log bisa disimpan.
11. Report bisa disimpan.
12. Integration setting bisa disimpan.
13. Notification log bisa disimpan.
14. Activity log bisa disimpan.
15. RLS melindungi data antar workspace.
16. Schema siap digunakan oleh Next.js app.
