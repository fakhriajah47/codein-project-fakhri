# 04 - API & Route Contract

## 1. Tujuan Dokumen

Dokumen ini menjelaskan kontrak API untuk aplikasi **Project Management**.

API digunakan untuk:

1. Authentication support.
2. Workspace management.
3. Project management.
4. Task management.
5. Milestone management.
6. AI features.
7. Report generation.
8. Notification integration.
9. Activity logging.
10. Dashboard analytics.

Aplikasi dibangun secara monolithic menggunakan Next.js, sehingga API dapat dibuat menggunakan:

* Next.js Route Handlers.
* Server Actions.
* Supabase client di server.
* Service layer modular.

---

# 2. API Design Principle

## 2.1 Server-First

Operasi yang menyentuh database dan secret wajib berjalan di server.

Contoh:

* Gemini API call.
* Telegram bot token.
* Discord webhook URL.
* Gmail OAuth token.
* Supabase service role key.
* Notification sending.

---

## 2.2 Consistent Response Format

Semua API harus memiliki format response yang konsisten.

### Success Response

```json
{
  "success": true,
  "message": "Project created successfully",
  "data": {}
}
```

### Error Response

```json
{
  "success": false,
  "message": "Failed to create project",
  "error": {
    "code": "VALIDATION_ERROR",
    "details": {}
  }
}
```

---

## 2.3 Validation First

Semua request body harus divalidasi menggunakan Zod sebelum diproses.

Contoh error:

```json
{
  "success": false,
  "message": "Invalid request body",
  "error": {
    "code": "VALIDATION_ERROR",
    "details": {
      "name": "Project name is required"
    }
  }
}
```

---

## 2.4 Role-Based Access

Setiap endpoint harus mengecek:

1. User sudah login.
2. User adalah member workspace.
3. Role user cukup untuk action tersebut.

Contoh:

* Viewer tidak boleh create task.
* Member tidak boleh manage integration.
* Owner bisa delete project.
* Manager bisa create report.

---

## 2.5 Activity Log by Default

Setiap action penting harus membuat activity log.

Contoh:

* Create project.
* Update task status.
* Generate AI report.
* Send notification.

# 3. Authentication Routes

Authentication utama menggunakan Supabase Auth.

## 3.1 Register

Route:

```txt
POST /api/auth/register
```

Request:

```json
{
  "email": "user@example.com",
  "password": "password123",
  "fullName": "Fakhri Rimbawan"
}
```

Response:

```json
{
  "success": true,
  "message": "Account created successfully",
  "data": {
    "userId": "uuid"
  }
}
```

Process:

1. Create user via Supabase Auth.
2. Create profile.
3. Return user data.

Acceptance Criteria:

* User bisa register.
* Profile otomatis dibuat.
* Password tidak disimpan manual di database aplikasi.

# 16. API Success Criteria

API dianggap berhasil jika:

1. Semua endpoint memiliki response format konsisten.
2. Semua request tervalidasi.
3. Semua endpoint protected.
4. Role-based access berjalan.
5. Semua action penting membuat activity log.
6. Project progress dihitung ulang saat task berubah.
7. AI route tidak membocorkan API key.
8. Notification route mencatat log sukses/gagal.
9. Gmail route bisa authorize dan send report.
10. Discord dan Telegram bisa test connection.
11. Dashboard route menampilkan data akurat.
12. Error message ramah dan mudah dipahami user.
13. API mudah dikembangkan karena service layer modular.
