# 08 - Implementation Tracker

## 1. Tujuan Dokumen

Dokumen ini adalah tracker implementasi untuk project **Project Management**.

Tracker ini dibuat agar proses development:

1. Terarah.
2. Modular.
3. Tidak bengkak di satu file.
4. Mudah dicek progress-nya.
5. Mudah dipresentasikan.
6. Bisa dieksekusi bertahap oleh developer atau AI agent.
7. Tetap sesuai PRD, database design, API contract, AI spec, integration spec, dan UI/UX blueprint.

---

# 2. Prinsip Eksekusi

## 2.1 Modular First

Setiap fitur harus dibuat dalam folder/module yang jelas.

Jangan menaruh semua logic di satu file besar.

Contoh yang benar:

```txt
features/projects/
├── components/
├── actions/
├── services/
├── validators/
├── types.ts
└── constants.ts
```

Contoh yang salah:

```txt
app/dashboard/page.tsx
```

Berisi semua logic project, task, kanban, AI, dan integration dalam satu file panjang.

---

## 2.2 Build in Small Vertical Slices

Implementasi dilakukan per alur fitur, bukan hanya per layer.

Contoh vertical slice:

```txt
Create Project Flow:
Database schema → Validator → Server action/API → UI form → Activity log → Success toast
```

Dengan cara ini, setiap fitur bisa dites dari ujung ke ujung.

---

## 2.3 Every Feature Must Have States

Setiap halaman/fitur wajib punya:

1. Loading state.
2. Empty state.
3. Success state.
4. Error state.
5. Unauthorized state jika perlu.

---

## 2.4 Every Important Action Must Be Logged

Action penting wajib masuk ke `activity_logs`.

Contoh:

1. Project dibuat.
2. Task dibuat.
3. Task status berubah.
4. AI analysis dibuat.
5. Report dikirim.
6. Notification gagal/sukses.

---

## 2.5 Keep UI Neobrutalism Consistent

Semua komponen utama harus mengikuti style:

1. Border hitam tebal.
2. Hard shadow.
3. Warna solid.
4. Typography bold.
5. Layout friendly.
6. UI tetap rapi dan profesional.

# 22. Final Implementation Rule

Setiap task development harus mengikuti rule berikut:

```txt
Before coding:
1. Baca dokumen terkait.
2. Pahami scope task.
3. Cek folder/module yang sesuai.
4. Buat file kecil dan spesifik.

During coding:
1. Jangan menumpuk logic di page component.
2. Pisahkan UI, service, validator, type, dan constants.
3. Gunakan TypeScript.
4. Gunakan Zod untuk validasi.
5. Gunakan server-side untuk secret.
6. Buat loading/error state.

After coding:
1. Test flow manual.
2. Cek console error.
3. Cek responsive.
4. Cek permission.
5. Update tracker.
```
