# 10 - README Setup Guide

# Project Management

**AI-powered project management command center for modern teams, founders, and project managers.**

Project Management adalah aplikasi project management berbasis web yang membantu tim mengelola project, task, progress, risiko, laporan, dan notifikasi otomatis dengan bantuan AI.

Aplikasi ini dibuat sebagai **portfolio-grade SaaS product** dengan pendekatan:

* Project management dashboard.
* AI task generator.
* AI risk analysis.
* AI executive summary.
* Kanban board.
* Workspace-based collaboration.
* Role-based access.
* Discord notification.
* Telegram alert.
* Gmail report sender.
* Modern Neobrutalism UI.

Project Management bukan sekadar aplikasi to-do list.

# 6. Required Environment Variables

Create `.env.local`:

```env
# App
APP_URL=http://localhost:3000
NEXT_PUBLIC_APP_URL=http://localhost:3000

# Supabase
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=

# Gemini AI
GEMINI_API_KEY=
GEMINI_MODEL=gemini-2.5-flash

# Discord
DISCORD_WEBHOOK_URL_DEFAULT=

# Telegram
TELEGRAM_BOT_TOKEN=
TELEGRAM_DEFAULT_CHAT_ID=

# Google / Gmail OAuth
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
GOOGLE_REDIRECT_URI=http://localhost:3000/api/integrations/gmail/callback
```

---

# 7. Environment Variable Rules

## Public Variables

Only variables with `NEXT_PUBLIC_` can be accessed from frontend.

Allowed:

```env
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
NEXT_PUBLIC_APP_URL=
```

## Secret Variables

Never expose these to frontend:

```env
SUPABASE_SERVICE_ROLE_KEY=
GEMINI_API_KEY=
TELEGRAM_BOT_TOKEN=
GOOGLE_CLIENT_SECRET=
DISCORD_WEBHOOK_URL_DEFAULT=
```

# 9. Supabase Setup

## 9.1 Create Supabase Project

1. Open Supabase dashboard.
2. Create new project.
3. Copy project URL.
4. Copy anon key.
5. Copy service role key.
6. Add them to `.env.local`.

## 9.2 Run Database Migration

If using Supabase CLI:

```bash
supabase db push
```

Or run SQL migration manually from:

```txt
supabase/migrations/
```

## 9.3 Enable Row Level Security

All app tables must have RLS enabled.

Required tables:

```txt
profiles
workspaces
workspace_members
projects
project_members
milestones
tasks
task_comments
task_attachments
activity_logs
ai_generations
reports
integration_settings
notification_logs
```

## 9.4 Setup Storage Bucket

Create bucket:

```txt
task-attachments
```

Recommended access:

* Private bucket.
* Access controlled by signed URL or workspace permission logic.

Can:

* Read selected project data.

# 12. AI Setup

## 12.1 Gemini API Key

Add:

```env
GEMINI_API_KEY=
```

AI calls must only happen from server-side route handlers or server actions.

## 12.2 AI Features

Routes:

```txt
POST /api/ai/generate-tasks
POST /api/ai/generated-tasks/save
POST /api/ai/risk-analysis
POST /api/ai/executive-summary
POST /api/ai/daily-focus
```

## 12.3 AI Rules

* Output must be valid JSON.
* Response must be validated with Zod.
* AI output must be reviewable.
* Generated tasks are not saved before user approval.
* Executive summary is not emailed before user preview.
* All generations are logged in `ai_generations`.
