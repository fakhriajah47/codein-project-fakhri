# 00 - Product Vision & Ideation

## Nama Project

**ProjectPilot AI**

## Tagline

**AI-powered project management command center untuk tim kecil, founder, dan agency yang ingin proyek berjalan rapi, cepat, dan terukur.**

## One-Liner

ProjectPilot AI adalah aplikasi project management modern berbasis web yang membantu tim mengelola project, task, deadline, prioritas, risiko, komunikasi, dan laporan progress secara terpusat dengan bantuan AI serta notifikasi otomatis ke Discord, Telegram, dan Gmail.

---

# 1. Latar Belakang

Banyak tim kecil, startup, digital agency, mahasiswa, dan founder memiliki masalah yang sama saat menjalankan project:

1. Task tercecer di chat.
2. Deadline tidak terpantau.
3. Progress sulit dibaca oleh leader.
4. Tim sering bingung prioritas.
5. Update project harus ditanya manual.
6. Risiko project baru sadar ketika sudah terlambat.
7. Laporan ke CEO/client masih dibuat manual.
8. Komunikasi project tersebar di WhatsApp, Discord, email, dan meeting.
9. File, komentar, task, dan keputusan tidak terdokumentasi dengan rapi.

Akibatnya, project terlihat sibuk tetapi tidak terukur. Banyak pekerjaan berjalan, tetapi leader tidak punya visibility yang jelas terhadap status sebenarnya.

ProjectPilot AI dibuat untuk menyelesaikan masalah tersebut dengan pendekatan:

* Semua project dalam satu dashboard.
* Semua task punya owner, deadline, status, prioritas, dan progress.
* AI membantu membaca risiko, menyusun ringkasan, dan memberi rekomendasi prioritas.
* Notifikasi otomatis dikirim ke channel yang biasa digunakan tim.
* CEO/founder/client bisa melihat executive dashboard tanpa harus membaca detail teknis.
* Dev/team member bisa fokus pada task harian.
* Project manager bisa memantau workload, bottleneck, dan timeline.

---

# 2. Visi Product

Menjadi platform project management berbasis AI yang simple, stylish, friendly, dan powerful untuk membantu tim kecil bekerja seperti tim profesional.

Project ini tidak hanya menampilkan fitur CRUD task, tetapi menunjukkan kemampuan membangun aplikasi SaaS modern dengan:

* Authentication.
* Role-based access.
* Workspace/project management.
* Task lifecycle.
* Activity log.
* Realtime dashboard.
* Notification automation.
* AI assistant.
* Integration system.
* Executive reporting.
* Modern UI/UX.

---

# 3. Positioning Product

## Bukan Sekadar To-Do App

ProjectPilot AI bukan aplikasi checklist biasa. Aplikasi ini bertindak sebagai:

1. **Project Command Center**
   Semua project, task, timeline, progress, dan risiko berada dalam satu tempat.

2. **AI Project Assistant**
   AI membantu membuat breakdown task, membaca risiko, membuat daily summary, dan memberi rekomendasi next action.

3. **Executive Dashboard**
   CEO/founder/client bisa melihat status project dalam bentuk visual, ringkas, dan profesional.

4. **Automation Hub**
   Update penting bisa otomatis dikirim ke Discord, Telegram, dan Gmail.

5. **Portfolio-Grade SaaS Product**
   Dibangun dengan struktur produk yang serius, UI modern, dan arsitektur scalable.

# 4. Target User

## 4.1 Primary User

### A. Founder / CEO / Business Owner

Kebutuhan:

* Ingin tahu semua project jalan atau macet.
* Tidak punya waktu membaca detail panjang.
* Butuh executive summary.
* Butuh tahu project mana yang berisiko.
* Butuh laporan profesional untuk decision making.

Pain Point:

* Harus tanya PM satu-satu.
* Update tersebar di chat.
* Sulit tahu siapa yang overload.
* Sulit tahu deadline mana yang kritis.

Value:

* Melihat semua project dalam satu dashboard.
* Mendapat AI executive summary.
* Melihat risk score.
* Mendapat update otomatis via email/Telegram/Discord.

---

### B. Project Manager / Team Lead

Kebutuhan:

* Membagi task ke tim.
* Memantau deadline.
* Melihat bottleneck.
* Mengirim update ke stakeholder.
* Menyusun report mingguan.

Pain Point:

* Terlalu banyak manual follow-up.
* Sulit memantau status real.
* Laporan progress makan waktu.
* Banyak task tanpa owner yang jelas.

Value:

* Bisa manage project, sprint, task, member, dan status.
* AI membantu generate breakdown task.
* AI membantu menyusun laporan.
* Notifikasi otomatis mengurangi follow-up manual.

---

### C. Developer / Designer / Team Member

Kebutuhan:

* Tahu task apa yang harus dikerjakan.
* Tahu prioritas harian.
* Bisa update progress dengan cepat.
* Bisa komentar dan attach file.
* Tidak ingin dashboard yang ribet.

Pain Point:

* Bingung prioritas.
* Task sering berubah tanpa dokumentasi.
* Komunikasi tersebar.
* Tidak tahu task yang paling urgent.

Value:

* Dashboard personal “My Work”.
* Task jelas: deadline, prioritas, deskripsi, checklist.
* AI daily focus.
* Notifikasi ketika ada update penting.

---

### D. Client / Stakeholder Viewer

Kebutuhan:

* Ingin tahu progress project.
* Tidak perlu masuk detail internal.
* Butuh laporan yang mudah dipahami.
* Butuh trust bahwa project berjalan.

Pain Point:

* Harus tanya progress manual.
* Bingung dengan istilah teknis.
* Tidak punya akses visual ke progress.

Value:

* Client portal sederhana.
* Lihat project status, milestone, dan progress.
* Dapat weekly summary via email.
* Tidak bisa melihat data internal sensitif.

# 5. Core Product Promise

ProjectPilot AI membantu tim menjawab 5 pertanyaan penting:

1. Project apa saja yang sedang berjalan?
2. Task mana yang harus dikerjakan hari ini?
3. Siapa sedang mengerjakan apa?
4. Deadline mana yang berisiko?
5. Apa next action terbaik agar project tetap on-track?

# 6. Unique Selling Point

## 6.1 AI Risk Radar

AI membaca data project dan memberi skor risiko berdasarkan:

* Task overdue.
* Banyak task high priority belum selesai.
* Progress lambat.
* Owner overload.
* Milestone dekat tetapi completion rendah.
* Tidak ada update activity dalam beberapa hari.

Output:

* Risk score.
* Risk reason.
* Suggested action.
* Escalation recommendation.

---

## 6.2 AI Task Breakdown

User bisa input brief project, lalu AI membuat:

* Epics.
* Milestones.
* Task list.
* Suggested assignee role.
* Priority.
* Estimated complexity.
* Acceptance criteria.

Contoh input:

“Buat website company profile untuk agency digital dengan halaman home, service, portfolio, about, contact.”

Output AI:

* Setup project structure.
* Design landing page.
* Build service section.
* Build portfolio CMS.
* Build contact form.
* Setup deployment.
* QA checklist.

---

## 6.3 AI Executive Summary

AI membuat laporan ringkas untuk CEO/client:

* Project health.

* Task selesai.

# 15. Product Principle

1. Simple first, powerful when needed.
2. AI membantu, bukan menggantikan user.
3. Dashboard harus cepat dipahami dalam 5 detik.
4. Semua task harus punya owner, status, dan deadline.
5. Semua update penting harus tercatat.
6. CEO melihat insight, tim melihat action.
7. Data harus aman per workspace.
8. Integrasi harus modular.
9. UI harus premium tetapi tidak rumit.
10. Portfolio harus terlihat production-ready.
