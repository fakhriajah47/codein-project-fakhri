# 07 - UI/UX Blueprint: Modern Neobrutalism Style

## 1. Tujuan Dokumen

Dokumen ini menjelaskan arahan UI/UX untuk aplikasi **ProjectPilot AI** dengan style utama:

```txt
Modern Neobrutalism
```

Style ini dipilih agar aplikasi terlihat:

1. Berani.
2. Fresh.
3. Stylish.
4. Friendly.
5. Berbeda dari dashboard SaaS biasa.
6. Tetap profesional untuk presentasi CEO.
7. Cocok sebagai portfolio yang memorable.

---

# 2. Product Design Direction

ProjectPilot AI harus terasa seperti:

```txt
A bold AI-powered project command center with playful structure, strong visual hierarchy, and professional workflow clarity.
```

Artinya:

* UI boleh playful, tetapi tidak berantakan.
* Visual boleh bold, tetapi informasi tetap mudah dibaca.
* Elemen boleh memiliki border tebal, tetapi layout harus tetap rapi.
* Warna boleh kontras, tetapi tidak membuat mata lelah.
* Dashboard harus tetap cepat dipahami.

---

# 3. Neobrutalism Style Definition

## 3.1 Ciri Utama

Style Neobrutalism pada ProjectPilot AI menggunakan:

1. Thick border.
2. Hard shadow.
3. Flat color.
4. Bold typography.
5. High contrast.
6. Slightly playful layout.
7. Geometric card.
8. Clear grid.
9. Minimal gradient.
10. Expressive icons.
11. Large CTA button.
12. Raw but intentional visual style.

---

## 3.2 Yang Harus Dihindari

Neobrutalism tidak berarti UI boleh asal.

Hindari:

1. Shadow terlalu berlebihan.
2. Warna terlalu ramai.
3. Typography terlalu banyak variasi.
4. Border tidak konsisten.
5. Layout berantakan.
6. Komponen terlalu childish.
7. Button terlalu besar semua.
8. Terlalu banyak animasi.
9. Terlalu banyak pattern.
10. Kontras yang membuat text sulit dibaca.

# 6. Border & Shadow Rules

## 6.1 Border

Default border:

```txt
2px solid #111111
```

Important card border:

```txt
3px solid #111111
```

Modal border:

```txt
3px solid #111111
```

Input border:

```txt
2px solid #111111
```

---

## 6.2 Shadow

Gunakan hard shadow khas Neobrutalism.

Default shadow:

```css
box-shadow: 4px 4px 0px #111111;
```

Hover shadow:

```css
box-shadow: 6px 6px 0px #111111;
transform: translate(-2px, -2px);
```

Pressed state:

```css
box-shadow: 2px 2px 0px #111111;
transform: translate(2px, 2px);
```

# 7. Typography

## 7.1 Font Recommendation

Gunakan font sans-serif modern dan bold.

Rekomendasi:

1. Inter.
2. Plus Jakarta Sans.
3. Space Grotesk.
4. Archivo.
5. Satoshi jika tersedia.

## 7.2 Font Style

### Heading

* Bold.
* Large.
* Clear.
* Tidak terlalu panjang.
* Gunakan line-height rapat tapi tetap terbaca.

### Body

* Medium readability.
* Jangan terlalu kecil.
* Gunakan warna charcoal atau black.

### Label

* Uppercase kecil untuk label status.
* Font weight 700.
* Letter spacing tipis.

# 9. Component Design System

## 9.1 Button

### Primary Button

Style:

```txt
Background: Electric Blue
Text: White
Border: 2px solid black
Shadow: 4px 4px black
Radius: 10px atau 12px
Font: Bold
```

Use Case:

* Create project.
* Generate with AI.
* Send report.
* Save changes.

### Secondary Button

Style:

```txt
Background: White
Text: Black
Border: 2px solid black
Shadow: 3px 3px black
```

Use Case:

* Cancel.
* View details.
* Filter.
* Open settings.

### Danger Button

Style:

```txt
Background: Coral Red
Text: Black/White
Border: 2px solid black
Shadow: 4px 4px black
```

Use Case:

* Delete.
* Remove member.
* Disable integration.

---

## 9.2 Card

Card default:

```txt
Background: White
Border: 2px solid black
Shadow: 4px 4px black
Radius: 16px
Padding: 20px - 24px
```

Important card:

```txt
Background: Bright Yellow / Mint / Blue
Border: 3px solid black
Shadow: 6px 6px black
```

Rule:

* Jangan semua card diberi warna kuat.
* Gunakan warna kuat hanya untuk highlight.
* Dashboard tetap dominan white/cream.

# 11. Interaction Design

## 11.1 Hover

Card hover:

```txt
Move slightly upward
Shadow becomes bigger
Cursor pointer
```

Button hover:

```txt
Translate -2px -2px
Shadow 6px 6px black
```

Button active:

```txt
Translate 2px 2px
Shadow 2px 2px black
```

---

## 11.2 Loading State

Gunakan skeleton dengan border hitam tipis.

Contoh:

```txt
Loading project dashboard...
```

Jangan hanya spinner. Untuk neobrutalism, loading bisa berupa card skeleton yang tetap punya border.

---

## 11.3 Empty State

Empty state harus friendly.

Contoh:

```txt
No projects yet.
Create your first project and let AI help you break it down.
```

CTA:

```txt
Create Project
```

Visual:

* Simple icon.
* Bordered illustration.
* Accent color.

---

## 11.4 Error State

Error harus jelas dan tidak teknis.

Contoh:

```txt
We couldn’t generate the AI summary.
```

CTA:

```txt
Try Again
```

# 12. Accessibility

## 12.1 Contrast

Karena neobrutalism memakai kontras tinggi, pastikan:

* Text selalu mudah dibaca.
* Jangan gunakan text kuning di background putih.
* Badge warna terang harus tetap punya text hitam.
* Button biru sebaiknya text putih.

## 12.2 Keyboard Navigation

* Semua button bisa difokuskan.
* Modal bisa ditutup dengan Escape.
* Form bisa diisi tanpa mouse.
* Focus ring harus jelas.

## 12.3 Font Size

Minimum body text:

```txt
14px
```

Recommended body text:

```txt
16px
```

Dashboard number:

```txt
32px - 48px
```

# 15. Page-Level Success Criteria

## 15.1 Landing Page

Berhasil jika:

1. User langsung paham value product.
2. Hero terlihat bold dan memorable.
3. CTA jelas.
4. Dashboard preview terlihat premium.
5. Tidak terlihat seperti template biasa.

## 15.2 Dashboard

Berhasil jika:

1. User bisa memahami project health dalam 5 detik.
2. At risk project langsung terlihat.
3. AI summary mudah ditemukan.
4. Data utama tidak tenggelam oleh dekorasi.

## 15.3 Kanban

Berhasil jika:

1. Task mudah dibaca.
2. Status task jelas.
3. Priority jelas.
4. User mudah update status.
5. Board tetap nyaman meskipun banyak task.

## 15.4 AI Assistant

Berhasil jika:

1. Fitur AI terlihat sebagai fitur unggulan.
2. Output AI mudah direview.
3. User tidak merasa AI memaksa.
4. CTA generate jelas.
5. Error AI ditangani dengan ramah.

## 15.5 Integrations

Berhasil jika:

1. User tahu integrasi mana yang aktif.
2. Test connection mudah dilakukan.
3. Error mudah dipahami.
4. Token/webhook tidak terekspos penuh.
5. Last delivery status terlihat.

# 16. Design Do and Don’t

## 16.1 Do

* Gunakan border hitam konsisten.
* Gunakan hard shadow secukupnya.
* Gunakan warna accent untuk highlight.
* Gunakan whitespace.
* Gunakan typography bold.
* Buat dashboard mudah dipindai.
* Buat empty state friendly.
* Buat CTA jelas.
* Buat AI feature terlihat spesial.
* Pastikan UI tetap profesional.

## 16.2 Don’t

* Jangan semua elemen diberi warna kuat.
* Jangan semua card terlalu besar.
* Jangan pakai gradient berlebihan.
* Jangan pakai shadow blur lembut terlalu banyak.
* Jangan membuat UI seperti game anak-anak.
* Jangan membuat dashboard terlalu ramai.
* Jangan menyembunyikan action penting.
* Jangan tampilkan secret integration penuh.
* Jangan gunakan font terlalu dekoratif.
* Jangan mengorbankan usability demi style.

---

# 17. Final UI Direction Statement

ProjectPilot AI menggunakan **Modern Neobrutalism** untuk menciptakan visual yang berani, fresh, dan memorable, namun tetap mempertahankan clarity, accessibility, dan professional workflow.

```txt
Simple to use.
Powerful to manage.
Bold enough to remember.
Professional enough to present.
```
