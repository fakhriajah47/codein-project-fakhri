# 15 - Validation Schema Spec

## 1. Tujuan Dokumen

Dokumen ini menjelaskan standar validasi data untuk aplikasi **ProjectPilot AI** menggunakan **Zod**.

Validasi diperlukan agar:

1. Input user aman.
2. Data yang masuk database konsisten.
3. API request tidak menerima payload sembarangan.
4. AI response bisa dipastikan valid.
5. Integration config tidak salah format.
6. Error message lebih ramah.
7. Developer tidak membuat validasi manual berulang-ulang.

---

# 2. Validation Principle

## 2.1 Validate at Boundary

Semua data harus divalidasi ketika masuk ke sistem.

Boundary utama:

1. Form submit.
2. API request body.
3. Server action input.
4. Query params.
5. AI response.
6. Integration callback.
7. File upload metadata.

---

## 2.2 Never Trust Client Input

Semua input dari frontend harus dianggap tidak aman sampai divalidasi di server.

Frontend validation boleh ada untuk UX, tetapi server validation tetap wajib.

---

## 2.3 Use Friendly Error Message

Error Zod harus diubah menjadi pesan yang mudah dipahami user.

Contoh buruk:

```txt
Expected string, received undefined
```

Contoh baik:

```txt
Project name is required.
```

---

## 2.4 Keep Schema Modular

Jangan taruh semua schema di satu file besar.

Gunakan folder:

```txt
lib/validators/
├── auth.schema.ts
├── workspace.schema.ts
├── project.schema.ts
```


## 3.1 UUID Schema

```ts
import { z } from "zod";

export const uuidSchema = z.string().uuid("Invalid ID format.");
```

## 3.2 Required String

```ts
export const requiredString = (fieldName: string, min = 1, max = 255) =>
  z
    .string()
    .trim()
    .min(min, `${fieldName} is required.`)
    .max(max, `${fieldName} is too long.`);
```

## 3.3 Optional String

```ts
export const optionalString = (max = 1000) =>
  z
    .string()
    .trim()
    .max(max, `Text is too long.`)
    .optional()
    .or(z.literal(""));
```

## 3.4 Date String

```ts
export const dateStringSchema = z
  .string()
  .regex(/^\d{4}-\d{2}-\d{2}$/, "Date must use YYYY-MM-DD format.");
```

## 3.5 Email Schema

```ts
export const emailSchema = z
  .string()
  .trim()
  .email("Please enter a valid email address.");
```

## 3.6 Pagination Schema

```ts
export const paginationSchema = z.object({
  page: z.coerce.number().int().min(1).default(1),
  limit: z.coerce.number().int().min(1).max(100).default(10)
});
```

# 16. Validation Usage Pattern

## 16.1 In Route Handler

```ts
export async function POST(req: Request) {
  try {
    const body = await req.json();
    const input = createProjectSchema.parse(body);

    const result = await projectService.createProject(input, context);

    return successResponse("Project created successfully", result);
  } catch (error) {
    return errorResponse(error);
  }
}
```

## 16.2 In Server Action

```ts
"use server";

export async function createProjectAction(values: unknown) {
  const input = createProjectSchema.parse(values);
  return await projectService.createProject(input, context);
}
```

## 16.3 In Frontend Form

```ts
const form = useForm<CreateProjectInput>({
  resolver: zodResolver(createProjectSchema),
  defaultValues: {
    name: "",
    description: "",
    priority: "medium"
  }
});
```

```ts
export type CreateProjectInput = z.infer<typeof createProjectSchema>;
export type UpdateProjectInput = z.infer<typeof updateProjectSchema>;
```

This prevents duplicate manual types.


# 19. Validation Success Criteria

Validation system dianggap berhasil jika:

```txt
[ ] Semua form punya schema.
[ ] Semua API route memvalidasi request body.
[ ] Semua query params divalidasi.
[ ] Semua AI response divalidasi.
[ ] Semua integration config divalidasi.
[ ] Semua email recipient divalidasi.
[ ] Semua file upload metadata divalidasi.
[ ] Error message ramah.
[ ] TypeScript type diambil dari Zod schema.
[ ] Tidak ada validasi manual berulang yang tidak perlu.
[ ] Tidak ada input user langsung masuk database tanpa validasi.
```

---

# 20. Final Validation Rule

```txt
If data enters the app, validate it.
If AI returns data, validate it.
If user sends a form, validate it.
If provider sends callback, validate it.
If database receives insert/update, validate it first.
```
