# 01 - Master PRD Project Management

## 1. Ringkasan Product

Project Management adalah aplikasi project management berbasis web untuk membantu founder, project manager, dan tim mengelola project secara terpusat dengan dukungan AI, dashboard analytics, dan integrasi notifikasi otomatis.

Aplikasi ini dibangun sebagai monolithic web app dengan Supabase sebagai backend platform. Frontend dan server-side logic berada dalam satu codebase agar mudah dikembangkan, mudah dideploy, dan cocok untuk kebutuhan portfolio profesional.

---

# 2. Tujuan Product

## 2.1 Tujuan Bisnis

1. Menunjukkan kemampuan membangun SaaS product modern.
2. Menjadi portfolio unggulan untuk dipresentasikan kepada CEO/founder.
3. Membuktikan kemampuan integrasi AI, database, authentication, notification, dan UI dashboard.
4. Menjadi template produk yang bisa dikembangkan menjadi SaaS komersial.
5. Menunjukkan pemahaman project management secara nyata, bukan hanya UI CRUD.

## 2.2 Tujuan User

1. Membuat project lebih rapi.
2. Membagi pekerjaan dengan jelas.
3. Memantau progress secara visual.
4. Mengurangi follow-up manual.
5. Mendeteksi risiko lebih awal.
6. Membuat laporan lebih cepat.
7. Menghubungkan project update ke channel komunikasi yang sudah digunakan tim.

# 6. Feature Requirement

### Description

User dapat register, login, logout, dan mengakses aplikasi sesuai session.

### Requirement

* Register dengan email dan password.
* Login dengan email dan password.
* Logout.
* Protected route.
* User profile basic.
* Session persistence.
* Redirect user ke dashboard setelah login.

### Acceptance Criteria

* User baru bisa membuat akun.
* User bisa login dengan credential valid.
* User tidak bisa akses dashboard tanpa login.
* User bisa logout.
* Session tetap aktif ketika refresh page.

## 6.8 AI Task Generator

### Output

* Suggested milestones.
* Task list.
* Task description.
* Priority.
* Suggested assignee role.
* Estimated complexity.
* Acceptance criteria.

### Requirement

* User klik “Generate with AI”.
* User memasukkan brief.
* Sistem memanggil Gemini API dari server.
* AI response harus berbentuk JSON/schema yang bisa diproses.
* User bisa review hasil AI sebelum menyimpan ke database.

### Acceptance Criteria

* AI output bisa disimpan menjadi task.
* User bisa edit hasil AI sebelum submit.
* Jika AI gagal, tampil error yang jelas.
* API key tidak pernah terekspos ke client.

# 8. Non-Functional Requirement

## 8.1 Performance

* Page dashboard load cepat.
* Query database harus efisien.
* Gunakan pagination untuk list besar.
* Gunakan loading skeleton.
* Hindari render data terlalu banyak sekaligus.

## 8.2 Security

* API key AI hanya di server.
* Webhook URL disimpan aman.
* Gmail token disimpan aman.
* Role-based access wajib.
* Workspace isolation wajib.
* Row Level Security wajib diterapkan.
* User tidak boleh mengakses workspace lain.

## 8.3 Reliability

* Integration failure tidak boleh membuat app crash.
* Semua notification send harus punya status log.
* AI failure harus punya fallback.
* Email failure harus bisa retry manual.

## 8.4 Scalability

* Struktur database harus mendukung multi-workspace.
* Integration dibuat modular.
* AI feature dibuat service terpisah dalam codebase.
* Report disimpan agar tidak selalu generate ulang.
