# 09 - Agent Execution Prompt

## 1. Tujuan Prompt

Prompt ini digunakan untuk menginstruksikan AI coding agent agar mengimplementasikan **Project Management** secara akurat, modular, rapi, profesional, dan sesuai seluruh dokumen PRD.

Project ini adalah portfolio-grade product yang akan dipresentasikan kepada CEO/founder, sehingga hasil akhir harus terlihat seperti produk serius, bukan demo asal jadi.

# 4. Anti-Bloat Rules for Agent

Agent wajib mengikuti aturan ini agar project tidak rusak dan tidak bengkak.

```txt
1. Jangan membuat file lebih dari 300-400 baris kecuali benar-benar perlu.
2. Jika file mulai panjang, split menjadi component/service/helper.
3. Jangan menaruh semua logic di page.tsx.
4. Jangan menaruh semua API logic di satu route besar.
5. Jangan membuat type berulang-ulang.
6. Jangan membuat komponen yang sebenarnya bisa reusable menjadi duplikat.
7. Jangan hardcode user ID.
8. Jangan hardcode workspace ID.
9. Jangan hardcode secret.
10. Jangan bypass RLS tanpa alasan.
11. Jangan menampilkan token/webhook penuh.
12. Jangan mengirim email tanpa preview.
13. Jangan menyimpan output AI tanpa validasi.
14. Jangan membuat UI berbeda style antar halaman.
15. Jangan mengabaikan loading/error/empty state.
```
