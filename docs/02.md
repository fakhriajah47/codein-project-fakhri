# 02 - Modular Breakdown & Technical Direction

## 1. Recommended Architecture

ProjectPilot AI dibangun sebagai **monolithic full-stack web application**.

## 1.1 Stack Utama

### Frontend

* Next.js App Router
* React
* TypeScript
* Tailwind CSS
* shadcn/ui
* Lucide Icons
* TanStack Query atau server actions pattern
* Zustand untuk client state ringan jika diperlukan
* React Hook Form
* Zod validation

### Backend Platform

* Supabase Auth
* Supabase Postgres
* Supabase Storage
* Supabase Realtime
* Supabase Row Level Security
* Supabase Edge Functions jika diperlukan untuk webhook/automation khusus

### Server-Side Logic

* Next.js Route Handlers
* Next.js Server Actions
* Supabase server client
* Service layer modular
* Environment variables di Vercel

### AI

* Gemini API dari Google AI Studio
* AI service layer di server
* Structured JSON output
* Prompt templates
* AI log table

### Integration

* Discord Webhook
* Telegram Bot API
* Gmail API OAuth
* Notification log
* Integration settings per workspace

### Deployment

* Vercel untuk web app
* Supabase hosted project untuk database/auth/storage
* Environment variables untuk secrets

# 4. Permission Model

## 4.1 Workspace Owner

* Full access semua data workspace.
* Bisa manage integration.
* Bisa invite/remove member.
* Bisa delete project.
* Bisa generate dan send report.

## 4.2 Manager

* Bisa manage project dan task.
* Bisa generate AI summary.
* Bisa kirim report.
* Tidak bisa hapus workspace.
* Tidak bisa mengubah owner.

## 4.3 Member

* Bisa melihat project yang dia ikuti.
* Bisa update task yang terkait.
* Bisa komentar.
* Bisa upload attachment.
* Tidak bisa manage integration.

## 4.4 Viewer

* Read-only.
* Hanya melihat project yang diberi akses.
* Tidak bisa melihat data internal tertentu.

# 5. AI Module Breakdown

## 5.1 AI Task Generator

Route:

```txt
POST /api/ai/generate-tasks
```

Input:

```json
{
  "projectName": "Website Company Profile",
  "projectDescription": "Membuat website company profile untuk agency digital",
  "deadline": "2026-07-01",
  "teamRoles": ["Designer", "Frontend Developer", "QA"],
  "complexity": "medium"
}
```

Output:

```json
{
  "milestones": [
    {
      "title": "Planning & Setup",
      "description": "Menyiapkan struktur awal project",
      "tasks": [
        {
          "title": "Define sitemap",
          "description": "Membuat struktur halaman website",
          "priority": "high",
          "suggestedRole": "Project Manager",
          "acceptanceCriteria": [
            "Sitemap mencakup semua halaman utama",
            "Sitemap disetujui oleh stakeholder"
          ]
        }
      ]
    }
  ]
}
```

Rules:

* Output wajib JSON valid.
* Tidak langsung insert ke database sebelum user approve.
* Hasil AI harus bisa diedit.
* Semua response AI disimpan ke ai_generations.

# 6. Notification System

## 6.1 Notification Service

Semua integrasi harus lewat satu service:

```txt
notification-service.ts
```

Tujuan:

* Menghindari logic tersebar.
* Memudahkan logging.
* Memudahkan retry.
* Memudahkan tambah provider baru.

## 6.2 Event Types

* project.created
* project.updated
* project.completed
* project.at_risk
* task.created
* task.assigned
* task.completed
* task.blocked
* task.overdue
* report.generated
* report.sent
* ai.risk_generated

## 6.3 Provider Rules

### Discord

Untuk update internal yang cukup detail.

Contoh:

```txt
[Task Completed]
Project: Website Company Profile
Task: Build Homepage
Completed by: Fakhri
Status: Done
```

### Telegram

Untuk alert singkat dan urgent.

Contoh:

```txt
Project Alert: Website Company Profile
Risk Level: High
Reason: 3 urgent tasks overdue.
Action: Please review dashboard.
```

### Gmail

Untuk report formal.

Contoh:

Subject:

```txt
Weekly Project Report - Website Company Profile
```

Body:

```txt
Hi CEO,

Here is the weekly update for Website Company Profile.

Overall status: At Risk
Progress: 72%

Completed:
- Homepage
- Service page
- Contact form

Risks:
- Deployment pending
- QA mobile pending

Next Actions:
- Complete QA
- Deploy staging
- Final client review
```

# 8. Environment Variables

```env
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=

SUPABASE_SERVICE_ROLE_KEY=

GEMINI_API_KEY=

DISCORD_WEBHOOK_URL_DEFAULT=

TELEGRAM_BOT_TOKEN=
TELEGRAM_DEFAULT_CHAT_ID=

GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
GOOGLE_REDIRECT_URI=

APP_URL=
```

Rules:

* NEXT_PUBLIC hanya untuk value yang aman di client.
* Secret key tidak boleh dipanggil dari browser.
* Service role key hanya digunakan di server.
* Gemini API key hanya digunakan di server.
* Gmail client secret hanya digunakan di server.
* Webhook URL sebaiknya disimpan terenkripsi atau minimal tidak pernah tampil penuh setelah disimpan.
